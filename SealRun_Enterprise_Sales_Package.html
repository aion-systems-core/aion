Part 1 — Technical Evidence Pack

All statements in this part are grounded in this repository. Line ranges are inclusive. Drift in line numbers should be resolved by searching the cited symbol.

1.1 Deterministic Execution and Capsule Hash

Blake3 over canonical semantic JSON

engine/src/capsule/hash.rs lines 24–48: deterministic_capsule_hash builds a HashPayload from prompt, seed, determinism_profile, canonical token_trace and events, canonical graph and why slices, and canonical drift JSON, then Blake3-hashes the canonical JSON string of that value. Field choice excludes capture-time noise by construction — no OS path or capsule timestamp fields in the payload struct.

Canonical JSON

engine/src/ai/canon.rs lines 11–24: canonicalize_value recursively sorts object keys (BTreeMap). Lines 48–69: canonical_capsule_bytes optionally strips version, execution_envelope, execution_environment, and evidence attestation flags from canonical comparison bytes.

Drift taxonomy

engine/src/ai/drift.rs lines 22–55: drift_between_runs compares tokens, semantic token_trace, canonical event order, causal graph edges only, and Why causal edges only; emits a DriftReport with sorted, deduplicated fields and details.

Runtime contract

engine/src/ai/runtime.rs lines 51–52: AiRuntime::run documents that the deterministic token stream depends on prompt, model, seed, and det only. Lines 84–88: events are passed through sort_events_deterministic before return.

1.2 Formal Replay Invariant (Abstract + Code + Tests)

TLA+ module

docs/formal/replay_invariant.tla lines 29–30: ReplayInvariant == (rId = 0) \/ (rId = cId) — after any replay, the replayed capsule id must match the executed id.

Shell helper

scripts/check_replay_invariant.sh lines 13–26: if tlc2 or java + TLA_TOOLS_JAR is available, TLC is executed with -deadlock. If not available, the script prints a skip message and exits 0 — does not fail the shell pipeline for missing TLC.

CI Scope
.github/workflows/ci.yml lines 17–21: CI runs cargo build --workspace and cargo test only. No step runs scripts/check_replay_invariant.sh in the default workflow.

Engine bridge

engine/src/replay/symmetry.rs lines 42–48: assert_formal_replay_invariant calls assert_replay_symmetry then requires deterministic_capsule_hash(original) == deterministic_capsule_hash(replayed).

Automated Rust tests

engine/tests/formal_replay.rs lines 7–20: formal_replay_invariant_holds rebuilds a capsule and expects success; formal_replay_invariant_breaks_on_tamper mutates a token and expects failure.

1.3 Syscall / IO Policy Surface

IO policy enum on determinism profile

core/src/contracts.rs lines 120–131: DeterministicIOPolicy is Strict | Audit | Deny, carried on DeterminismProfile (io_policy field, lines 95–114).

Syscall whitelist and evaluation

engine/src/syscall/policy.rs lines 7–37: SyscallName covers read, write, open, stat, clock_gettime (with fstat/lstat normalised to Stat in parse). Lines 60–87: evaluate_syscall returns Err for disallowed calls; should_block maps Strict and Deny to a hard-block flag.

First-class capsule events

engine/src/ai/trace.rs lines 27–41: Event::SyscallCaptured and Event::PolicyViolation variants carry syscall audit payloads as first-class capsule event stream entries.

1.4 Cross-Machine Replay

Machine fingerprint

kernel/src/envelope.rs lines 14–22: MachineFingerprint stores cpu_features, os_version, kernel_version, hostname_hash, binary_hash. Lines 29–30: capture_machine_fingerprint is documented as best-effort with no network I/O.

Capsule embedding

engine/src/ai/model.rs lines 9–37: ExecutionEnvironment pairs machine_fingerprint and runtime_fingerprint; optional on AICapsuleV1 as execution_environment.

Validator

engine/src/replay/cross_machine.rs lines 6–41: machine_fingerprint_tolerant_equal compares CPU/OS/kernel only. validate_cross_machine_replay requires matching determinism, matching capsule Blake3 hash, matching token trace and event stream, and strict runtime_fingerprint equality when both sides carry execution_environment, with tolerant machine fingerprint comparison (hostname/binary path excluded).

What SealRun is
SealRun is a deterministic AI execution layer. Inference-style runs are sealed into capsules — structured JSON artefacts carrying token trace, event stream, causal graph, Why payloads, drift summaries, and a linear evidence chain. A Blake3 digest over canonical JSON of selected semantic fields (deterministic_capsule_hash, hash.rs 24–48) provides a stable, portable capsule identity. Capsules can be replayed, compared field-by-field via a structured drift report, and linked into a rolling SHA-256 evidence chain with optional Ed25519 signing through the C ABI. A minimal TLA+ abstract module ships under docs/formal/ (optional TLC via scripts/check_replay_invariant.sh; not part of default CI). Default CI (.github/workflows/ci.yml) runs cargo test, including engine/tests/formal_replay.rs, which exercises assert_formal_replay_invariant on live capsules — distinct from the TLA+ abstraction.

Evidence Chain — Structure and Verification
Chain structure
EvidenceChain is a linear list of EvidenceRecord values. Each record carries a leaf_digest (SHA-256 of the step bytes), a rolling payload_digest (SHA-256 of parent|leaf), and a parent_digest pointer. Optional boolean fields formal_replay_invariant_ok and cross_machine_replay_ok record post-hoc attestation results.
core/src/evidence/chain.rs lines 7–27
Seal
seal_run builds the rolling SHA-256 chain over run result, policy, determinism profile, and schema version artefacts. Starting digest label: GENESIS.
core/src/evidence/proof.rs lines 11–45
Verify
verify_linear recomputes each rolling digest from leaf_digest and parent_digest; returns Err on any mismatch. Tamper test in governance_evidence.rs confirms rejection when leaf_digest is mutated.
core/src/evidence/proof.rs lines 47–61 · governance_evidence.rs lines 77–89
Ed25519 Signing (optional)
C ABI exports aion_evidence_generate_keypair, aion_evidence_sign, and aion_evidence_verify_with_key. Sign/verify roundtrip and tamper rejection tested in governance_evidence.rs lines 13–27.
