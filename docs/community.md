# Community and support

This page defines public support channels for SealRun users and contributors.

## At a glance

- Use public issue channels for product and docs feedback.
- Keep security-sensitive disclosures private.
- Include deterministic command evidence when reporting issues.

- GitHub issues for bug reports and feature requests.
- GitHub discussions (recommended) for design and usage questions.
- Security reports should be routed privately through repository security policy.

## CLI surface

When opening issues, include relevant outputs:

```bash
sealrun --version
sealrun doctor
```

## Enterprise-readiness

Support quality improves when reports include machine-readable contract outputs rather than free-form descriptions only.
