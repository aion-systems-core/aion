# Feedback survey

## Purpose

Structured feedback channel description; technical contracts remain in [OS Contract Spec](os_contract_spec.md).

This survey collects structured feedback for deterministic SealRun product evolution.

## At a glance

- Focus on determinism, replay, governance, and integrations
- Inputs support product, docs, and enterprise-readiness improvements
- Prefer reproducible examples and command-level references

Please provide feedback via GitHub issues with:

1. Primary use case
2. Determinism/replay pain points
3. Desired integrations
4. SDK/API improvements requested

## CLI surface

When reporting issues, include relevant command outputs, for example:

```bash
sealrun doctor
sealrun governance status
sealrun tests strategy
```

## Enterprise-readiness

Feedback quality is highest when it references deterministic artifacts and machine-readable contract outputs.
